## 내 답변

React에서 비동기 요청은 보통 useEffect 안에서 처리합니다. 컴포넌트가 처음 렌더링되거나 특정 상태값이 바뀌었을 때 API 요청을 보내고, 응답을 받아 useState로 화면을 갱신하는 방식입니다. 이때 useEffect의 의존성 배열은 언제 이 로직을 다시 실행할지 결정하므로 매우 중요합니다. 배열이 비어 있으면 마운트 시 한 번만 실행되고, 값이 들어 있으면 해당 값이 변경될 때만 다시 실행됩니다. 의존성 배열을 잘못 설정하면 요청이 반복 실행되거나, 반대로 필요한 시점에 요청이 실행되지 않는 문제가 생길 수 있습니다. 면접에서는 “데이터 fetch 시점 제어와 불필요한 재호출 방지”가 핵심이라고 말하면 좋습니다.

## 모범 답변

React에서 비동기 요청을 처리할 때 useEffect 훅을 사용합니다. 예를 들어, 컴포넌트가 처음 마운트될 때 API 요청을 보내고, 응답을 useState로 저장하여 화면을 갱신합니다. 의존성 배열은 useEffect가 언제 실행될지를 결정하는 중요한 요소입니다. 만약 배열이 비어 있다면, useEffect는 컴포넌트가 처음 렌더링될 때 한 번만 실행됩니다. 반면, 배열에 특정 상태나 props가 들어가면 그 값이 변경될 때마다 useEffect가 재실행됩니다. 잘못된 의존성 배열 설정은 불필요한 API 호출을 유발하거나, 예상치 못한 시점에 요청이 이루어지지 않게 만들 수 있습니다. 따라서 의존성 배열을 적절히 설정하는 것이 중요합니다. 예를 들어, [userId]를 의존성으로 넣으면 userId가 변경될 때마다 해당 사용자의 데이터를 다시 가져올 수 있습니다.
