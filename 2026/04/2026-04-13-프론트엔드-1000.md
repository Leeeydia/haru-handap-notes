# React의 Virtual DOM이 동작하는 원리를 설명해주세요.

> **카테고리**: 프론트엔드 | **난이도**: 중 | **점수**: 85/100

---

## 내 답변

React의 Virtual DOM은 실제 DOM을 바로 수정하지 않고, 메모리상 가상 트리를 먼저 비교해 변경된 부분만 찾아 실제 DOM에 반영하는 방식입니다. 상태가 바뀌면 새로운 Virtual DOM을 만들고 이전 것과 비교(diff)한 뒤, 필요한 최소 변경만 적용(reconciliation)해 불필요한 렌더링을 줄이고 UI 업데이트 성능을 높입니다.

## 모범 답변

React의 Virtual DOM은 실제 DOM을 직접 수정하는 대신, 메모리 내에서 가상의 DOM 트리를 생성하여 이를 통해 UI 업데이트를 효율적으로 관리합니다. 상태가 변경되면 새로운 Virtual DOM을 생성하고, 기존의 Virtual DOM과 비교(diffing)하여 변경된 부분만을 찾아냅니다. 이 프로세스를 통해 필요한 최소한의 변경만 실제 DOM에 적용(reconciliation)하여, 불필요한 렌더링을 줄이고 성능을 최적화합니다. 예를 들어, 리스트의 한 항목이 변경될 경우, 전체 리스트를 다시 렌더링하는 대신, 해당 항목만 업데이트할 수 있습니다.
