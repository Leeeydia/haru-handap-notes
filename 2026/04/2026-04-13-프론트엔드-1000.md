## 내 답변

Virtual DOM is an in-memory representation of the real DOM. When state changes, React creates a new Virtual DOM tree and diffs it against the previous one using a reconciliation algorithm. Only the changed nodes are updated in the real DOM, minimizing expensive DOM operations and improving rendering performance.

## 모범 답변

The Virtual DOM is an in-memory representation of the real DOM that allows React to optimize updates. When a component's state changes, React generates a new Virtual DOM tree and compares it to the previous version using a process called 'diffing'. This involves checking each node for differences and determining which parts of the tree have changed. Once the differences are found, React updates only the affected nodes in the real DOM, which significantly reduces the number of costly DOM manipulations and improves rendering performance. This reconciliation process is crucial for efficient UI updates in React applications.
