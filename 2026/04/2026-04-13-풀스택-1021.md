# React에서 상태 관리를 위해 useState와 useReducer 훅을 사용할 때 각각의 차이점과 사용 사례를 설명해 주세요.

> **카테고리**: 풀스택 | **난이도**: 중 | **점수**: 85/100

---

## 내 답변

useState는 단순한 상태를 다룰 때 적합합니다. 문자열, 숫자, boolean, 입력값처럼 상태 변화가 비교적 단순하고 업데이트 방식이 직관적일 때 많이 사용합니다. 코드가 짧고 이해하기 쉬워서 작은 컴포넌트에 유리합니다.

반면 useReducer는 상태 구조가 복잡하거나, 여러 상태가 서로 연결되어 함께 바뀌는 경우에 적합합니다. reducer 함수로 상태 변경 로직을 분리하고, dispatch로 액션을 전달해 상태를 관리하므로 로직을 체계적으로 정리할 수 있습니다. 특히 폼 상태가 많거나, 로딩/성공/실패처럼 상태 전환이 명확한 경우에 유용합니다.

정리하면 useState는 간단한 상태 관리, useReducer는 복잡한 상태 변화와 명확한 액션 기반 관리에 적합합니다. 면접에서는 “상태 변경 로직의 복잡도”를 기준으로 구분해서 설명하면 좋습니다.

---

## AI 피드백

### 내용 완성도
상태 관리에 대한 전반적인 이해를 잘 설명하였으나, useReducer의 사용 사례에 대한 구체적인 예시가 부족합니다.

### 답변 구조
답변이 명확하게 두 부분으로 나뉘어 있어 이해하기 쉬웠습니다. 그러나 각 훅의 설명을 더 균형 있게 다루면 좋을 것 같습니다.

### 표현/말투
표현이 간결하고 명확하여 좋았으나, 약간의 기술적인 용어 사용이 더해지면 전문성이 더해질 것입니다.

### 구체성
상태의 복잡성과 액션 기반 관리를 언급하여 적절한 구체성을 보여주었습니다. 그러나 구체적인 사례나 예시를 추가하면 더욱 좋습니다.

## 모범 답변

useState는 단순한 상태를 다루기에 적합하며, 주로 문자열, 숫자, boolean과 같은 간단한 상태 변화에 사용됩니다. 예를 들어, 사용자 입력값을 관리할 때 유용합니다. 반면, useReducer는 상태의 구조가 복잡하거나 여러 상태가 서로 연결되어 있을 때 적합합니다. 예를 들어, 여러 폼 필드의 상태를 동시에 관리하거나 로딩, 성공, 실패 상태를 명확히 구분할 때 유용합니다. useReducer는 reducer 함수를 통해 상태 변경 로직을 분리하고, dispatch를 통해 액션을 전달하여 상태를 관리합니다. 따라서 상태 변경 로직의 복잡도에 따라 useState와 useReducer를 적절히 선택해 사용하는 것이 중요합니다.
